## Root Cause

**StateObject.pManagedObject is never initialized**

Looking at InitClientDLL disassembly:
- Sets StateObject.vtable at 0x62852478
- Sets StateObject.pManagedObject to NULL at 0x628524a8

```asm
62852498: mov ecx, [0x6299996c]  ; Check if already initialized
6285249e: test ecx, ecx
628524a0: je skip_call
628524a2: mov edx, [ecx]
628524a4: push 0x1
628524a6: call [edx]             ; Call if exists
628524a8: mov [0x6299996c], 0x0  ; <-- ALWAYS sets to NULL
```

**Conclusion**: client.dll itself initializes pManagedObject to NULL!

This means:
- Either launcher.exe should provide valid pManagedObject through SetMasterDatabase
- OR client.dll expects to create this itself later
- OR we're missing a registration step

---

## SetMasterDatabase Analysis

From client.dll disassembly, SetMasterDatabase accepts:
```c
void SetMasterDatabase(void* pMasterDatabase);
```

If pMasterDatabase != NULL, it treats it as linked-list node.

**Hypothesis**: The structure passed by launcher contains pointers that should affect StateObject initialization.
